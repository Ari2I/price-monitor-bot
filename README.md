# Мониторинг цен конкурентов — Telegram-бот

Telegram-бот на Python, который отслеживает цены товаров на
произвольных сайтах (обычных интернет-магазинах и сайтах с
динамической JS-подгрузкой цены), хранит историю цен и присылает
отчёты — как по расписанию, так и по запросу, в виде текстового
сообщения и Excel-файла.

## Возможности

- **Универсальный парсер цены**: сначала пробует обычный HTTP-запрос
  (быстро, подходит для большинства интернет-магазинов), и если цена
  не найдена в статическом HTML — автоматически переключается на
  рендеринг страницы через headless-браузер (Playwright) для сайтов
  с динамической подгрузкой данных.
- **Диалоговое добавление товара** через Telegram (FSM): название →
  ссылка → CSS-селектор цены → нужен ли принудительный режим
  браузера.
- **История цен** в базе данных (SQLite по умолчанию, легко
  переключить на PostgreSQL — см. `DATABASE_URL`), с возможностью
  посмотреть изменения во времени командой `/history`.
- **Отчёты по расписанию**: раз в заданный интервал бот сам проверяет
  цены всех товаров каждого пользователя и присылает сводку в
  Telegram (текстом и, опционально, Excel-файлом).
- **Отчёты по команде**: `/report` — текстовая сводка прямо сейчас,
  `/excel` — выгрузка текущего среза цен в файл `.xlsx`.
- Многопользовательский режим: каждый пользователь бота видит и
  отслеживает только свои товары.

## Источник данных

Бот работает с произвольными сайтами, которые укажет пользователь —
это не единый API, а универсальный парсер по CSS-селектору,
который пользователь указывает вручную при добавлении товара.
Никакие механизмы обхода защит сайтов (ротация IP, подмена cookie,
антибот-байпас) не используются и не реализовывались — бот отправляет
обычные запросы с браузерным заголовком `User-Agent`, как это делает
любой обычный браузер.

## Установка

Требуется Python 3.10 или новее.

```bash
pip install -r requirements.txt
playwright install chromium
```

Команда `playwright install chromium` нужна один раз — она
скачивает headless-браузер, который используется для сайтов с
динамической подгрузкой цены. Без этого шага бот всё равно будет
работать с обычными (статическими) сайтами, но не сможет
обрабатывать JS-сайты.

## Настройка

1. Скопируйте `.env.example` в `.env`:

   ```bash
   cp .env.example .env
   ```

2. Получите токен бота у [@BotFather](https://t.me/BotFather) в
   Telegram и укажите его в `.env` как `BOT_TOKEN`.
3. При необходимости настройте интервал автоматической проверки
   (`CHECK_INTERVAL_MINUTES`) и другие параметры — все они описаны
   комментариями прямо в `.env.example`.

## Запуск

```bash
python main.py
```

## Использование в Telegram

| Команда | Что делает |
|---|---|
| `/start` | Приветствие и список команд |
| `/add` | Пошаговое добавление нового товара для отслеживания |
| `/list` | Список отслеживаемых товаров с последней известной ценой |
| `/remove <id>` | Удалить товар из отслеживания |
| `/report` | Проверить цены прямо сейчас и получить текстовую сводку |
| `/excel` | Проверить цены прямо сейчас и получить Excel-файл |
| `/history <id>` | История изменения цены конкретного товара |
| `/cancel` | Отменить текущий диалог (например, добавление товара) |

### Как найти CSS-селектор цены на сайте

1. Откройте страницу товара в браузере (Chrome/Firefox).
2. Кликните правой кнопкой мыши по цене → «Просмотреть код»
   (Inspect).
3. В открывшемся коде найдите класс или `id` элемента, в котором
   написана цена, например `<span class="price">1 299 руб.</span>`.
4. При добавлении товара в бота укажите `span.price` (или `.price`,
   если известен только класс).

Если бот не может найти цену обычным запросом, он автоматически
попробует загрузить страницу через headless-браузер — в большинстве
случаев дополнительная настройка не требуется. Принудительный режим
браузера (вопрос "сайт подгружает цену через JavaScript?" при
добавлении) имеет смысл указывать только для ускорения работы,
когда вы точно знаете, что сайт всегда требует JS.

## Формат Excel-отчёта

Файл содержит одну таблицу со следующими колонками: `Название`,
`Ссылка`, `Текущая цена`, `Предыдущая цена`, `Изменение`,
`Изменение, %`, `Последняя проверка`.

## Тестирование

Тесты проверяют логику приложения (разбор цены, работу с БД, сборку
Excel-отчёта, оркестрацию статика/браузер-фолбэк) без обращения к
реальным сайтам или Telegram API — все сетевые вызовы подменяются
моками:

```bash
python -m unittest discover -s tests -v
```

## Известные ограничения

- Точность извлечения цены зависит от корректности указанного
  CSS-селектора — при изменении вёрстки сайта селектор может
  потребовать обновления.
- Автоматический фолбэк на Playwright запускает полноценный
  headless-браузер, что заметно медленнее обычного HTTP-запроса —
  для большого числа товаров с динамическими сайтами это стоит
  учитывать при выборе интервала проверки.
- Бот не обходит капчи и активные антибот-системы — для сайтов с
  такой защитой потребуется отдельная доработка, которая намеренно
  не входит в объём этого проекта.
- Планировщик рассылает отчёт всем пользователям бота, у которых
  есть отслеживаемые товары; часовые пояса пользователей не
  учитываются — время проверки одинаково для всех.

## Чего в проекте намеренно нет

Как и в предыдущих проектах портфолио, здесь отсутствуют функции
ротации/смены IP-адреса, подмены cookie и обхода антибот-защит
сторонних сайтов — такие механизмы выходят за рамки задачи учебного
и демонстрационного проекта.
